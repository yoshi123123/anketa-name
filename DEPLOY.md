# Деплой на Render.com (бесплатно)

Render — самый простой бесплатный хостинг для Node.js. Карта не нужна.

## Шаг 1. GitHub

1. Создай аккаунт на <https://github.com>, если ещё нет.
2. Создай новый **приватный** репозиторий, например `anketa-forum`.
3. Загрузи туда содержимое этой папки. Самый простой способ — через интерфейс GitHub:
   - Открой репозиторий → **Add file → Upload files**
   - Перетащи `server.js`, `package.json`, `README.md`, `.env.example`, `DEPLOY.md` и папку `public/`
   - **НЕ загружай** файл `.env` (если ты его уже создал) и `db.sqlite` — там пароли и локальные данные
   - Внизу нажми **Commit changes**

## Шаг 2. Render

1. Зайди на <https://render.com> и зарегистрируйся через GitHub.
2. На дашборде нажми **New +** → **Web Service**.
3. Подключи свой репозиторий `anketa-forum` (Render запросит доступ — разреши).
4. Заполни форму:
   - **Name:** `anketa-forum` (или любое — это станет частью URL)
   - **Region:** Frankfurt (ближе к РФ/СНГ) или любой
   - **Branch:** `main`
   - **Build Command:** `npm install`
   - **Start Command:** `node server.js`
   - **Instance Type:** **Free**

## Шаг 3. Persistent disk (чтобы БД не терялась)

Бесплатный план Render перезагружает контейнер при простое — **без диска БД будет обнуляться**.

1. На странице сервиса слева — вкладка **Disks** → **Add Disk**
2. **Name:** `data`
3. **Mount Path:** `/var/data`
4. **Size:** 1 GB (минимум, бесплатно)
5. Нажми **Save**

## Шаг 4. Переменные окружения

На вкладке **Environment** → **Add Environment Variable** добавь:

| Key | Value |
|---|---|
| `OWNER_USERNAME` | твой логин главного админа, например `glom` |
| `OWNER_PASSWORD` | сложный пароль, минимум 10 символов |
| `JWT_SECRET` | длинная случайная строка (см. ниже как сгенерировать) |
| `DB_PATH` | `/var/data/db.sqlite` |
| `NODE_ENV` | `production` |

Сгенерировать `JWT_SECRET` — выполни на своём компьютере в терминале:
```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```
Скопируй результат в значение переменной.

## Шаг 5. Запуск

Нажми **Manual Deploy** → **Deploy latest commit**. Подожди 2–3 минуты — увидишь зелёную надпись «Live».

URL сайта будет вида `https://anketa-forum-xxxx.onrender.com` — он показан вверху страницы сервиса.

## Шаг 6. Первый вход

Открой URL → нажми **Войти** → введи `OWNER_USERNAME` и `OWNER_PASSWORD` из переменных. Готово.

## Замечания про free-план Render

- Если на сайт никто не заходит ~15 минут, контейнер засыпает. Первый запрос после простоя — холодный (10–30 сек), дальше быстро. Это нормально для бесплатного плана.
- Persistent disk сохраняет БД даже при засыпании.
- Для постоянно работающего сайта нужен платный план ($7/мес), но для начала бесплатного хватит.

## Обновление кода

Когда захочешь поменять код — отредактируй файлы в GitHub-репозитории и нажми commit. Render автоматически задеплоит новую версию.

## Если что-то сломалось

На странице сервиса вкладка **Logs** покажет ошибки. Самые частые:
- **«Cannot find module»** — забыл `npm install` в Build Command
- **«EACCES: /var/data»** — не подключил Disk на шаге 3
- **БД сбрасывается** — `DB_PATH` не указывает на `/var/data/...`
